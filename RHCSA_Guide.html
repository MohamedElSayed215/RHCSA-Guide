01 Configure Network & Hostname Network ▶
  • IP Address: 172.25.250.11/24
  • Default Gateway: 172.25.250.254
  • DNS Server: 172.25.250.254
  • Hostname: serverb.lab.example.com
1
Show existing connections to find the connection name
nmcli con show
2
Configure IP, gateway, DNS, and switch to static
nmcli con mod "System eth0" \
  ipv4.addresses 172.25.250.11/24 \
  ipv4.gateway 172.25.250.254 \
  ipv4.dns 172.25.250.254 \
  ipv4.method manual
3
Apply the connection and set the hostname
nmcli con up "System eth0"
hostnamectl set-hostname serverb.lab.example.com
4
Verify
ip addr show eth0
hostnamectl status
💡
ipv4.method manual = static IP. Replace "System eth0" with your actual connection name from nmcli con show.
02 Configure YUM Repository Repository ▶
  • File: /etc/yum.repos.d/rhel9.repo
  • Two repos: AppStream + BaseOS
  • Base URL: http://classroom.example.com/content/rhel9.0/x86_64/dvd/
  • gpgcheck=0
1
Create the repo file with both repositories
cat > /etc/yum.repos.d/rhel9.repo << 'EOF'
[AppStream]
name=AppStream
baseurl=http://classroom.example.com/content/rhel9.0/x86_64/dvd/AppStream
enabled=1
gpgcheck=0

[BaseOS]
name=BaseOS
baseurl=http://classroom.example.com/content/rhel9.0/x86_64/dvd/BaseOS
enabled=1
gpgcheck=0
EOF
2
Verify both repos are listed
dnf repolist
💡
gpgcheck=0 is required in the exam environment — no GPG key is available.
03 SELinux — Allow Port 82 for httpd SELinux ▶
  • Allow SELinux to use port tcp/82 for httpd
  • Change httpd Listen port to 82
  • Open firewall for port 82/tcp
  • Enable and start httpd
1
Add port 82 to SELinux http_port_t context
semanage port -a -t http_port_t -p tcp 82
semanage port -l | grep http
2
Change httpd to listen on port 82
sed -i 's/^Listen 80/Listen 82/' /etc/httpd/conf/httpd.conf
grep Listen /etc/httpd/conf/httpd.conf
3
Open firewall and start httpd
firewall-cmd --permanent --add-port=82/tcp
firewall-cmd --reload
systemctl enable --now httpd
⚠️
NEVER run setenforce 0 in the exam. Fix SELinux properly with semanage. Order matters: SELinux → httpd.conf → firewall → systemctl.
04 Create Users & Groups Users ▶
  • Create group: admin
  • harry and natasha — secondary group: admin
  • sarah — shell: /sbin/nologin
  • Password for all: password
1
groupadd admin
useradd -G admin harry
useradd -G admin natasha
useradd -s /sbin/nologin sarah
2
Set passwords
echo 'password' | passwd --stdin harry
echo 'password' | passwd --stdin natasha
echo 'password' | passwd --stdin sarah
3
Verify
id harry
id natasha
id sarah
💡
-G (uppercase) = supplementary group. -s /sbin/nologin prevents interactive login for sarah.
05 Collaborative Directory /common/admin Permissions ▶
  • Create directory: /common/admin
  • Group ownership: admin
  • Permissions: 2770 — SGID + rwx for owner/group
1
mkdir -p /common/admin
chgrp admin /common/admin
chmod 2770 /common/admin
2
Verify — look for the s in group execute bit
ls -ld /common/admin
💡
SGID (2xxx): new files inside inherit the group admin automatically. The s replaces the group x bit.
06 Configure AutoFS AutoFS ▶
  • Install autofs
  • Mount point: /localhome
  • production5 → servera:/user-homes/production5
  • NFS options: rw,sync
1
dnf install -y autofs
2
Add master map entry
echo "/localhome  /etc/auto.localhome" >> /etc/auto.master
3
Create the indirect map file
echo "production5  -rw,sync  servera.lab.example.com:/user-homes/production5" \
  > /etc/auto.localhome
4
systemctl enable --now autofs
💡
auto.master = which mount points to manage. auto.localhome = per-user NFS details. autofs mounts on-demand and unmounts after idle.
07 Schedule Cron Job for harry Cron ▶
  • User: harry
  • Schedule: every day at 12:30
  • Command: /bin/echo "hello"
1
Edit harry's crontab (opens vi)
crontab -u harry -e
2
Add this line — minute hour day month weekday
30 12 * * * /bin/echo "hello"
3
Verify
crontab -u harry -l
💡
Format: minute hour day-of-month month day-of-week. 30 12 * * * = 12:30 every day. Use -u harry to edit as root.
08 Configure NTP Client (chronyd) NTP ▶
  • NTP server: classroom.example.com
  • Config file: /etc/chrony.conf
  • Enable and start chronyd
1
echo "server classroom.example.com iburst" >> /etc/chrony.conf
systemctl enable --now chronyd
2
Verify sync
chronyc sources -v
💡
RHEL 9 uses chrony (not ntpd). iburst = fast initial sync. The ^* in sources output means currently synced.
09 Find Files Owned by sarah Find ▶
  • Find all files owned by user sarah
  • Copy them to /root/find.user/
1
mkdir -p /root/find.user
find / -user sarah -exec cp {} /root/find.user/ \; 2>/dev/null
2
Verify
ls /root/find.user/
💡
-exec cp {} /dest/ \; runs cp for each found file. 2>/dev/null suppresses "permission denied" errors from system directories.
10 Search Text in /etc/passwd Text ▶
  • Search for word home in /etc/passwd
  • Save output to /root/search.txt
1
grep "home" /etc/passwd > /root/search.txt
cat /root/search.txt
💡
> redirects stdout to a file (overwrites). >> appends. Always verify with cat after.
11 Create User with Specific UID Users ▶
  • Username: alies
  • UID: 1326
1
useradd -u 1326 alies
id alies
💡
-u (lowercase) sets the UID. Always verify with id username after creation.
12 Create Compressed Archive Archive ▶
  • Source: /var/tmp
  • Archive: /root/test.tar.gz
  • Compression: gzip
1
tar -czf /root/test.tar.gz /var/tmp
2
Verify contents
tar -tzf /root/test.tar.gz | head
💡
c=create, z=gzip, f=filename. The filename MUST come immediately after f. To extract: tar -xzf file.tar.gz.
13 Build Container Image Containers ▶
  • Registry: utility.lab.example.com
  • Credentials: admin / redhat
  • Build image named monitor
  • Do NOT modify the Containerfile
1
podman login utility.lab.example.com -u admin -p redhat
2
podman build -t monitor http://classroom.example.com/Containerfile
podman images
⚠️
RHEL 9 exam uses podman — NOT docker. They work similarly but docker is not installed.
14 Run Container as Systemd Service Containers ▶
  • Container name: ascii2pdf from image monitor
  • Volume: /opt/files → /opt/incoming:Z
  • Volume: /opt/processed → /opt/outgoing:Z
  • Service name: container-ascii2pdf
  • Enable linger for user student
1
Run the container
podman run -d --name ascii2pdf \
  -v /opt/files:/opt/incoming:Z \
  -v /opt/processed:/opt/outgoing:Z \
  monitor
2
Generate systemd service file
mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user
podman generate systemd --name ascii2pdf --files --new
3
Enable linger (service survives logout) and start
loginctl enable-linger student
systemctl --user daemon-reload
systemctl --user enable --now container-ascii2pdf
4
Verify
systemctl --user status container-ascii2pdf
podman ps
💡
loginctl enable-linger is critical — without it the service stops when you log out. :Z on volumes fixes SELinux context for bind mounts.
15 Set Default umask for natasha Permissions ▶
  • New files: r-------- (400)
  • New directories: r-x------ (500)
  • Make it permanent in .bashrc
1
Calculate: file base 666 − 400 = umask 266. Dir base 777 − 500 = umask 277. Use 0266.
echo "umask 0266" >> /home/natasha/.bashrc
grep umask /home/natasha/.bashrc
💡
umask subtracts permissions. 0266 gives files=400, dirs=511 (but dirs lose write, giving 500). Add to .bashrc so it's permanent.
16 Set Password Expiry Policy Security ▶
  • Passwords expire after 20 days
  • Applies to newly created users
  • File: /etc/login.defs
1
sed -i 's/^PASS_MAX_DAYS.*/PASS_MAX_DAYS   20/' /etc/login.defs
grep PASS_MAX_DAYS /etc/login.defs
💡
/etc/login.defs affects new users only. For existing users, use chage -M 20 username. The exam usually only asks for new users.
17 Configure Passwordless sudo for admin Sudo ▶
  • Group admin can run all commands
  • No password required (NOPASSWD)
  • File: /etc/sudoers.d/admin
1
echo "%admin ALL=(ALL) NOPASSWD: ALL" > /etc/sudoers.d/admin
chmod 440 /etc/sudoers.d/admin
visudo -c
💡
%admin means the group admin. chmod 440 is required — visudo rejects wrong permissions. Always validate with visudo -c.

Tasks 18–20
18 Create LVM Logical Volume LVM ▶
  • PV: /dev/sdc
  • VG: datastore with PE size 8 MiB
  • LV: database — 50 extents
  • Filesystem: ext3
  • Persistent mount: /mnt/database
1
Create physical volume and volume group
pvcreate /dev/sdc
vgcreate -s 8MiB datastore /dev/sdc
2
Create logical volume (50 extents = 50 × 8MiB = 400 MiB)
lvcreate -l 50 -n database datastore
3
Format and mount
mkfs.ext3 /dev/datastore/database
mkdir -p /mnt/database
echo '/dev/datastore/database /mnt/database ext3 defaults 0 0' >> /etc/fstab
mount -a
4
Verify
df -h /mnt/database
vgdisplay datastore
💡
-l 50 = 50 extents (logical). -L 400M = absolute size. Check PE size with vgdisplay. Order: pvcreate → vgcreate → lvcreate → mkfs → fstab → mount.
19 Resize Logical Volume to 100 Extents LVM ▶
  • Extend database LV to 100 extents total
  • Resize the filesystem to use the new space
1
Extend the LV
lvextend -l 100 /dev/datastore/database
2
Resize the ext3 filesystem (must unmount first)
umount /mnt/database
resize2fs /dev/datastore/database
mount -a
3
Verify
df -h /mnt/database
lvdisplay /dev/datastore/database
💡
ext3/ext4: unmount → resize2fs → mount. XFS: xfs_growfs /mnt/database (no unmount needed). Know which filesystem you have!
20 Configure Tuned Profile Tuned ▶
  • Install and start tuned
  • Get the recommended profile
  • Apply it
1
dnf install -y tuned
systemctl enable --now tuned
2
Get recommendation and apply it
tuned-adm recommend
tuned-adm profile virtual-guest
3
Verify active profile
tuned-adm active
💡
tuned-adm recommend outputs the best profile for the system. Apply whatever it suggests — in a VM it's usually virtual-guest.
